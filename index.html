Equipo de Desarrollo

Roadmap de Skills de IA

Conocimientos y ejercicios obligatorios que cada desarrollador del equipo debe dominar.

1

Fundamentos y herramientas base

  • Instalar, usar y entender OpenClaw.
  • Instalarlo en una VPS personal e interactuar con él vía Telegram y WhatsApp.
  • Entender la diferencia entre usarlo vía API versus el plan ya suscrito de la cuenta.
2

Bases vectoriales y RAG

  • Conceptos: embeddings, similitud coseno, chunking, indexación, retrieval.
  • Ejercicio: cargar un corpus, generar embeddings y almacenarlos en una DB vectorial (Qdrant, Pinecone, pgvector, Chroma o Weaviate).
  • Construir un chat que conversa con esa base vectorial (RAG end-to-end).
3

MCP & Skills

  • Crear un servidor MCP propio.
  • MCP vs Skills: entender que MCP es la capa/protocolo para exponer contexto, recursos y herramientas al modelo, mientras que Skills definen capacidades, instrucciones y flujos de uso para resolver tareas específicas.
  • Convivencia: aprender cómo un Skill puede indicar cuándo usar un servidor MCP y cómo aprovechar correctamente las tools y recursos que ese MCP expone.
  • Implementar herramientas (tools) dentro del servidor MCP, no dentro del Skill.
  • Tool específica en el MCP: desde un chat, traducir la intención a SQL, consultar una tabla y retornar los datos en lenguaje natural.
4

OpenAI SDK + Tools + MCP con frontend

  • Instalar y hacer un ejercicio con OpenAI SDK.
  • Usar tools y MCP, y desplegarlo.
  • Construir un frontend con un chat básico que pueda interactuar con el modelo (que usa tools y MCP).
  • Queda a criterio del desarrollador qué harán las tools y el MCP.
5

Seguridad en aplicaciones con IA

  • Prompt injection y defensas.
  • Sanitización de outputs antes de ejecutar SQL/código.
  • PII handling y guardrails.
  • Riesgos específicos de tools que tocan DBs (read-only, scoping, rate limits).
6

Selección y comparación de modelos

  • Cuándo usar cada modelo: Opus vs Sonnet vs Haiku vs gpt-codex-5.3 / 5.4 / 5.5 vs modelos chinos.
  • Trade-offs costo / latencia / calidad.
  • Fine-tuning vs RAG vs prompt engineering: cuándo aplicar cada uno.
7

Calculadora de costo de tokens

  • Construir una calculadora que estime cuánto se gastaría en un chat usando la API de OpenAI u otros modelos.
  • Calcular tokens de input, tokens de output y los tokens del contexto que viajan en cada llamada.
  • Calcular los tokens consumidos al hacer embeddings en la carga inicial hacia la base de datos vectorial.
  • Calcular los tokens consumidos en cada consulta a la DB vectorial, ya que la API de embeddings se usa tanto para la carga inicial como para las consultas.
8

Asistente de voz sobre el chat existente

  • Al chat previamente creado, agregarle un botón de micrófono y convertir la interfaz en un asistente de voz que utilice el mismo backend.
  • Sumar el uso de voz mediante OpenAI (Coral) o ElevenLabs.
  • Esto permitirá interactuar con el bot vía chat o voz.
  • La voz debe ser realtime.
